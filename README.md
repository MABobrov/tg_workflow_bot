# Telegram Workflow Bot (проекты/статусы)

Бот переводит «менеджер пишет в чат свободным текстом» в структурированные проекты с карточками и задачами.

## 1) Быстрый запуск (локально)

1. Установите Python 3.11+
2. Создайте и заполните `.env`:
   ```bash
   cp .env.example .env
   ```
   Укажите как минимум:
   - `BOT_TOKEN`
   - `ADMIN_IDS`
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запуск:
   ```bash
   python -m app.main
   ```

## 2) Первичная настройка

1. Каждый сотрудник должен написать боту `/start`.
2. Каждый сотрудник присылает свой `/id` администратору.
3. Администратор назначает роли:
   ```text
   /setrole @manager_username manager
   /setrole @pasha_username rp
   /setrole @serga_username td
   /setrole @accounting_username accounting
   /setrole @installer_username installer
   /setrole @ceo_username gd
   ```
   Для нескольких ролей одному пользователю:
   ```text
   /setrole @username manager,rp
   /addrole @username td
   /removerole @username rp
   /setrole @username none
   ```
   Также поддерживается старый формат по `telegram_id`.

### Work chat (групповой чат уведомлений)
Добавьте бота в рабочий групповой чат. Бот сам отправит сообщение с `chat_id` и готовой командой.
Затем выполните в личке бота:
```text
/setworkchat -1001234567890
```

### Дефолтные исполнители (если несколько РП/ТД/бухгалтеров)
```text
/setdefaults rp=@pasha_username td=@serga_username acc=@accounting_username driver=@driver_username tinter=@tinter_username gd=@ceo_username
```
Также поддерживаются значения по `telegram_id`.
При передаче `@username` в `/setdefaults` бот сохраняет никнейм как дефолтное значение.

### Полезные админ-команды
```text
/admin_help       # подробная инструкция администратора прямо в боте
/users            # список заведённых сотрудников (или /employees)
/stats            # статистика использования бота
/addrole          # добавить роль пользователю
/removerole       # убрать роль у пользователя
/workchat         # диагностика привязки рабочего чата
/workchat_test    # тест отправки в рабочий чат
/resyncsheets     # полная пересинхронизация таблиц Projects/Tasks в читаемом формате
```

## 3) Основные сценарии

### Менеджер
- **Проверить КП/Запросить документы** → единый старт флоу, внутри выбор шага:
  - «Передать КП РП» (запрос КП)
  - «Запросить документы/счёт»
- После создания откройте карточку проекта: в ней есть кнопки всех следующих шагов
  (`Оплата`, `Док. / ЭДО`, `Проблема`, `Счёт End`) и список задач по этому проекту.
- **💰 Оплата поступила** → задача ТД на подтверждение оплаты
- **📄 Док. / ЭДО** → задача бухгалтерии
- **🏁 Счёт End** → финальное подтверждение (РП + админ)
- **🚨 Срочно ГД** → срочная задача Ген.Диру
- **🆘 Проблема / вопрос** → задача РП

### РП/ТД/Бухгалтерия
- **📥 Входящие задачи** → список задач
- Внутри задачи доступны действия: взять/завершить/отклонить
- Для подтверждения оплаты есть отдельные кнопки: «Оплата подтверждена» / «Нужна доплата»
- Бухгалтерия: **📨 Менеджеру (Имя)** → запрос недостающей информации менеджеру

### Ген.Дир
- **📥 Входящие задачи** → входящие срочные вопросы и поручения
- **🚨 Срочно ГД** → можно эскалировать срочный вопрос в поток ГД

### Монтажники
- **📝 Отчёт за день** → ежедневный отчёт (уходит РП + в рабочий чат)
- **🆘 Проблема / вопрос** → задача РП
- **✅ Счёт ОК** → подтверждение завершения монтажа (РП + админ)
- **🚨 Срочно ГД** → срочная эскалация Ген.Диру

## 4) Интеграция с Google Sheets (опционально)

1. Создайте Google Spreadsheet и возьмите его ID (из URL).
2. Создайте Service Account в Google Cloud и скачайте JSON-ключ.
3. Поделитесь таблицей с e-mail сервисного аккаунта (как с пользователем).
4. В `.env` включите:
   - `SHEETS_ENABLED=true`
   - `GSHEET_SPREADSHEET_ID=...`
   - `GOOGLE_SERVICE_ACCOUNT_FILE=./secrets/google-sa.json`
   - или `GOOGLE_SERVICE_ACCOUNT_JSON=...` (можно вставить JSON целиком или base64)

Таблица будет содержать листы:
- `Projects` — проекты
- `Tasks` — задачи

## 5) Интеграция amoCRM / Kommo (опционально)

В `.env` включите:
- `AMOCRM_ENABLED=true`
- `AMOCRM_BASE_URL=https://<subdomain>.amocrm.ru` (или `.kommo.com`)
- Вариант 1 (быстрый): только `AMOCRM_ACCESS_TOKEN`
- Вариант 2 (полный, с автообновлением): `AMOCRM_REFRESH_TOKEN` + OAuth2 параметры (`CLIENT_ID`, `CLIENT_SECRET`, `REDIRECT_URI`)

Важно:
- При режиме только с `AMOCRM_ACCESS_TOKEN` бот запустится и будет работать, пока токен действителен.
- Если токен истечёт, для автообновления нужно добавить `AMOCRM_REFRESH_TOKEN` и OAuth2-поля.

В текущей логике бот автоматически создаёт лид в amoCRM при создании проекта из сценариев:
- «Проверить КП/Запросить документы» → шаг «Запросить документы/счёт»
- «Проверить КП/Запросить документы» → шаг «Передать КП РП»

Создание дублей защищено: если у проекта уже есть `amo_lead_id`, повторный лид не создаётся.

## 6) Docker (опционально)

```bash
docker compose up -d --build
```

Данные SQLite сохраняются в `./data`.
Роли/сотрудники не теряются при обычном перезапуске контейнера (`docker compose restart` или `docker compose up -d`),
так как БД хранится в `./data/bot.sqlite3` и смонтирована в контейнер как `/app/data/bot.sqlite3`.
Не используйте удаление папки `./data`, если хотите сохранить сотрудников и роли.
Если включены Google Sheets через `GOOGLE_SERVICE_ACCOUNT_FILE=./secrets/google-sa.json`,
папка `./secrets` должна существовать на хосте и монтироваться в контейнер как `/app/secrets`
(в `docker-compose.yml` это уже настроено).
