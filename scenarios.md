## 1. Общая схема жизненного цикла проекта

```mermaid
stateDiagram-v2
    state "Лид amoCRM" as s0
    state "Запрос документов" as s1
    state "Запрос КП" as s2
    state "Счёт отправлен" as s3
    state "Ожидание оплаты" as s4
    state "Оплата заявлена" as s5
    state "В работе" as s6
    state "Заказ материалов" as s7
    state "Доставка" as s8
    state "Монтаж" as s9
    state "Тонировка" as s10
    state "Закрывающие документы" as s11
    state "Архив" as s12

    [*] --> s0 : Новый лид
    s0 --> s1 : Менеджер взял лид — запрос документов
    s0 --> s2 : Менеджер взял лид — запрос КП

    [*] --> s1 : Менеджер создаёт проект (документы)
    [*] --> s2 : Менеджер создаёт проект (КП)

    s1 --> s3 : РП подготовил документы
    s2 --> s3 : РП подготовил КП

    s3 --> s4 : Документы/счёт отправлены клиенту
    s4 --> s5 : Менеджер сообщил об оплате
    s5 --> s6 : ТД подтвердил оплату
    s5 --> s4 : ТД — нужна доплата

    s6 --> s7 : РП заказал материалы
    s7 --> s8 : РП создал заявку на доставку
    s6 --> s8 : РП создал заявку на доставку
    s8 --> s9 : Водитель доставил
    s6 --> s9 : Монтаж начат
    s9 --> s10 : РП создал заявку на тонировку
    s10 --> s11 : Тонировка завершена
    s9 --> s11 : Монтаж завершён (без тонировки)
    s11 --> s12 : Менеджер завершил проект
```

## 2. Менеджер

Роль: «Менеджер» | Паша и другие менеджеры по продажам.

### 2.1 Запрос документов / счёта

**Триггер:** Кнопка «Проверить КП/Запросить документы» или `/docs`

```mermaid
sequenceDiagram
    participant М as Менеджер
    participant Бот as Telegram-бот
    participant РП as Руководитель проектов
    participant Чат as Рабочий чат

    М->>Бот: Нажал «Проверить КП/Запросить документы»
    Бот->>М: Шаг 1: Название проекта?
    М->>Бот: «Ленина 45»
    Бот->>М: Шаг 2: Адрес?
    М->>Бот: Адрес
    Бот->>М: Шаг 3: Клиент?
    М->>Бот: ООО «Рога»
    Бот->>М: Шаг 4: Сумма?
    М->>Бот: 150000
    Бот->>М: Шаг 5: Дедлайн?
    М->>Бот: 25.03.2026
    Бот->>М: Шаг 6: Размеры/ТЗ?
    М->>Бот: 3x2.5м, закалённое 10мм
    Бот->>М: Шаг 7: Комментарий?
    М->>Бот: Просят срочно
    Бот->>М: Шаг 8: Фото/файлы? (Готово/Пропустить)
    М->>Бот: [фото] → Готово

    Бот->>Бот: Создаёт проект (Запрос документов)
    Бот->>Бот: Создаёт задачу «Запрос документов» → РП
    Бот->>Бот: Привязывает лид amoCRM (если есть)
    Бот->>РП: 🟢 Новый запрос: документы + вложения
    Бот->>Чат: 🟢 Новый запрос: документы
    Бот->>М: ✅ Проект создан
```

**Создаётся:** Проект (статус «Запрос документов»), Задача («Запрос документов», назначена РП, дедлайн 24ч).
**Уведомления:** РП (личное + вложения), Рабочий чат.
**amoCRM:** Если у менеджера есть невыкупленный лид — привязывается к проекту. Иначе создаётся новый лид.

## 4. ТД — Технический директор

Роль: «ТД» | Сергей (технический директор).

### 4.1 Подтверждение оплаты

**Триггер:** Кнопка «✅ Подтверждение оплат» или встроенные кнопки в задаче

```mermaid
sequenceDiagram
    participant ТД as Тех. директор
    participant Бот as Telegram-бот
    participant М as Менеджер
    participant РП as РП

    Note over ТД: Получает задачу «Подтверждение оплаты»
    ТД->>Бот: Открывает задачу
    alt Оплата подтверждена
        ТД->>Бот: «✅ Оплата подтверждена»
        Бот->>Бот: Задача → Выполнена
        Бот->>Бот: Проект → В работе
        Бот->>М: ✅ Оплата подтверждена
        Бот->>РП: ✅ Оплата подтверждена
    else Нужна доплата
        ТД->>Бот: «⚠️ Нужна доплата»
        Бот->>Бот: Задача → Отклонена
        Бот->>Бот: Проект → Ожидание оплаты
        Бот->>М: ⚠️ Нужна доплата
        Бот->>РП: ⚠️ Нужна доплата
    end
```

## 9. ГД — Генеральный директор

Роль: «ГД» | Сергей (совмещает с ТД).

ГД получает:

- Все срочные задачи («Срочно ГД»)
- Эскалации невзятых лидов (после 15 мин) — с кнопками назначения менеджеру
- Доступ к поиску проектов и входящим задачам

Основное меню ГД совпадает с меню ТД (т.к. Сергей = ТД + ГД).

## 10. Лиды из amoCRM

### 10.1 Автоматическое получение новых лидов

```mermaid
sequenceDiagram
    participant CRM as amoCRM
    participant Поллер as Поллер лидов (фон)
    participant БД as База данных
    participant Чат as Рабочий чат
    participant М as Менеджеры

    loop Каждые 30 секунд
        Поллер->>CRM: Запрос новых лидов (по дате создания)
        CRM-->>Поллер: Список новых лидов
        Поллер->>БД: Сохранить новые лиды
        Поллер->>Чат: 🔔 Новый лид + кнопка «🙋 Взять»
    end
```

---

### 10.2 Взятие лида менеджером

```mermaid
sequenceDiagram
    participant М as Менеджер
    participant Бот as Telegram-бот
    participant БД as База данных
    participant CRM as amoCRM
    participant Чат as Рабочий чат

    М->>Бот: Нажал «🙋 Взять лид в работу» (в чате)
    Бот->>БД: Забрать лид (атомарно)
    alt Успешно
        БД-->>Бот: Успех
        Бот->>Чат: ✅ Лид взят менеджером (кнопка убрана)
        Бот->>CRM: Назначить ответственного менеджера
        Бот->>М: (личное) ✅ Вы взяли лид «Название»
    else Уже взят
        БД-->>Бот: Занят
        Бот->>М: Лид уже взят другим менеджером
    end
```

---

### 10.3 Эскалация невзятого лида (15 мин)

```mermaid
sequenceDiagram
    participant Поллер as Поллер лидов (фон)
    participant БД as База данных
    participant РП as РП
    participant ГД as ГД

    loop Каждые 30 секунд
        Поллер->>БД: Найти неэскалированные лиды (старше 15 мин)
        БД-->>Поллер: Список невзятых лидов
        Поллер->>РП: ⚠️ Лид не взят 15+ мин!<br/>Кнопки выбора менеджера
        Поллер->>ГД: ⚠️ Лид не взят 15+ мин!<br/>Кнопки выбора менеджера
        Поллер->>БД: Пометить лид как эскалированный
    end
```

### 10.4 Назначение лида РП/ГД


## 11. Универсальные действия с задачами

Эти кнопки появляются при открытии любой задачи (встроенная клавиатура).

```mermaid
stateDiagram-v2
    state "Открыта" as s1
    state "В работе" as s2
    state "Выполнена" as s3
    state "Отклонена" as s4

    [*] --> s1 : Задача создана
    s1 --> s2 : ⏳ Взять в работу
    s1 --> s4 : ❌ Отклонить
    s2 --> s3 : ✅ Завершить
    s2 --> s4 : ❌ Отклонить

    note right of s1
        Кнопки — Взять / Отклонить
    end note
    note right of s2
        Кнопки — Завершить / Отклонить
    end note
    note right of s3
        Кнопка — Открыть (просмотр)
    end note
    note right of s4
        Кнопка — Открыть (просмотр)
    end note
```

Специальные кнопки для задач типа «Подтверждение оплаты»:

| Кнопка | Действие | Статус задачи | Статус проекта |
|--------|----------|---------------|----------------|
| ✅ Оплата подтверждена | «Оплата ОК» | → Выполнена | → «В работе» |
| ⚠️ Нужна доплата | «Нужна доплата» | → Отклонена | → «Ожидание оплаты» |

Специальная кнопка для задач «Заказ профиля» / «Заказ стекла» / «Заказ материалов»:

| Кнопка | Действие |
|--------|----------|

## 13. Админ-команды

Доступны администраторам бота.

| Команда | Описание |
|---------|----------|
| `/setrole @user role1,role2` | Установить роли пользователю |
| `/addrole @user role` | Добавить роль |
| `/removerole @user role` | Убрать роль |
| `/users` или `/employees` | Список пользователей + управление (роли, блокировка, задачи) |
| `/stats` | Статистика: пользователи, проекты, задачи, активность |
| `/setworkchat chat_id` | Установить рабочий чат для уведомлений |
| `/workchat_test` | Тест отправки в рабочий чат |
| `/resyncsheets` | Пересинхронизация Google Sheets |
| `/setdefaults rp=@user td=@user ...` | Установить исполнителей по умолчанию по ролям |
| `/setsetting key value` | Установить произвольную настройку |
